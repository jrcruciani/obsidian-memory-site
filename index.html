How it works

Simple architecture. High leverage.

The system is intentionally low-tech: a disciplined folder structure, a few key files, and clear loading rules that any capable AI can follow.

01

Store context in Markdown

Put identity, projects, people, vocabulary, tasks, and durable decisions into human-readable files inside your Obsidian vault.

02

Load only what matters

Use a master file plus a memory index so the AI reads the right files first and does not drown itself in unnecessary context.

03

Maintain it over time

The guide includes update and review routines so the system stays sharp instead of becoming another graveyard of stale notes.

Honest positioning

What this is, and what it is not.

What it is

  • A documented system for persistent AI context
  • A reusable structure for tasks, projects, people, and decisions
  • A way to reduce re-explaining yourself across sessions
  • An open-source guide you can adapt to your own vault

What it is not

  • Not autonomous memory injection into every AI product
  • Not mind reading, not AGI, not a replacement for judgment
  • Not a promise that every assistant will behave identically
  • Not zero-maintenance; the system works because you review it

FAQ

The obvious questions

Do I need Obsidian plugins to use this?

No. The core system is just files and folders in an Obsidian vault. Obsidian is the home, not a technical dependency.

Does this only work with Claude?

No. Claude Code is a strong fit, but the system is intentionally vendor-neutral. Any assistant that can read relevant files can use it.

Will this make an AI remember everything automatically?

No. It gives you a disciplined way to provide durable context. The quality still depends on the tool, the prompt, and what files you load.

Why not use a dedicated memory product instead?

You can. This project exists for people who want open formats, inspectable data, and a workflow they fully control.
